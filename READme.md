## Part 1 — Responsive Layout

You have been given a simple HTML page with basic CSS already applied. The student who built it used static pixel values to size everything — which means the layout breaks on different screen sizes.

Your job is to update the existing CSS rules so the page responds to any screen size. You do not need to rewrite the HTML. Open the CSS file and make your changes to the rules that are already there.

---

### Task 1 — Make the image responsive

Right now the image is sized with a static pixel value. Update the CSS so that the image takes up **50% of its container** no matter what screen size it is viewed on.


---

### Task 2 — Make the text responsive

The heading and paragraph text are currently using static pixel values for font size. Update the CSS so that the text scales with the screen size.


---

### Task 3 — Make the main section fill the screen

We want the main section to always take up the full height of the screen regardless of how tall the browser window is.

Write a rule so that the main section fills the full viewport height.

## Rubric

### LG 7.0 — Dynamic Units

| Level | What this looks like |
|-------|-------------|
| **4 — Exceeds** | All three tasks are complete and use the correct unit for each one. The student also made at least one additional change beyond the tasks — for example applying a responsive unit to another element not mentioned in the tasks — and that change improves the layout. |
| **3 — Meets** | All three tasks are complete. The correct unit is used in each task — `%` for the image, `vw` for text, `vh` for the main section. The page responds visibly when the browser window is resized. |
| **2 — Approaching** | At least one task is complete with the correct unit. Other tasks are attempted but use the wrong unit or are not fully applied. |
| **1 — No attempt** | No changes have been made to the CSS file beyond the setup code. Static pixel values remain throughout. |
